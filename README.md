# NekoDSL

NekoDSL is a shader language and compiler that compiles a custom, modern syntax down to SPIR-V binaries.

## Code Sample

`sample.neko`
```neko
import "stl.neko";
expose namespace stl;

vertex_main :: (pos : vec3) $ neko_position : vec4 {
    return vec4(pos, 1.0f);
}

fragment_main :: () $ neko_color : vec4 {
    return vec4(1.0f, 0.0f, 0.0f, 1.0f);
}
```

## Output Formats
* **SPIR-V Binary** (.spv)
* **SPIR-V ASM** (Human-readable assembly)

## Features
- **Custom Syntax**: `let mut` variable declarations, functional pipeline, decorators.
- **Control Flow**: Structured `if/else` and `while` loop support.
- **Built-in Optimizations**: Integrates `SPIRV-Tools-opt` for `-O1` (size) and `-O2` (performance) optimization passes.
- **Advanced Types & Operations**: Supports generic vector types (`vec2`, `vec3`, `vec4`), texture samplers, and Uniform Buffer Objects (UBO).
- **Module System**: Provides `import` and `expose import` directives natively for structured, component-based code sharing.
- **Interpolation Modes**: Easily apply `flat`, `noperspective`, or `centroid` qualifiers on varying logic limits.

## Getting Started

These instructions will get you a copy of the project up and running on your local machine for development and testing purposes.

### Prerequisites

* CMake >= 3.15
* A C++17 compatible compiler (e.g., MinGW, GCC, MSVC)
* Python >= 3 (Required by SPIRV-Tools and tooling)

### Checkout

```sh
git clone <repo-url>
cd NekoDSL
git submodule update --init --recursive
```

### Build

NekoDSL uses CMake for its build system. To configure and build the project:

```sh
mkdir build
cd build
cmake ..
cmake --build . -j4
```

The following options are available to control which targets are generated by CMake:

| Option | Description | Default |
| --- | --- | --- |
| NEKO_COMPILER | Generate `nekoc` target | ON |
| NEKO_DOCS | Generate documentation target (Requires Sphynx) | OFF |
| NEKO_GEN_STL | Generate STL files (Requires Python) | OFF |
| NEKO_GRAMMAR | Build grammar (Requires Bison and re2c) | OFF |
| NEKO_INSTALL | Generate install targets | OFF |
| NEKO_TESTS | Generate test target | OFF |

To build tests explicitly, configure with `-DNEKO_TESTS=ON`.

### Update Source

```sh
git pull
git submodule sync
git submodule update --init --recursive
```

## Running the Compiler

The primary compiler executable is `nekoc` (or `nekoc.exe` on Windows), typically output inside `build/bin/`.

To compile a `.neko` source file to `.spv`:

```sh
./build/bin/nekoc sample.neko -o sample.spv
```

### Compiler Options:
- `-o <file>` : Specify the output file path.
- `-O1`, `-O2` : Apply SPIR-V optimizations (size, performance respectively).
- `-S` : Output the shader as SPIR-V Assembly (human-readable) instead of binary.
- `-m <dir>` : Provide optional module search directories for resolving `import "..."` targets.

## Testing

NekoDSL provides a suite of unit/end-to-end tests to verify continuous parsing & correct AST/SPIR-V logic generation.

If configured with `-DNEKO_TESTS=ON`:
1. Navigate to the build directory.
2. Run standard CMake tests:
```sh
ctest --output-on-failure
```
Alternatively, just run the compiled GoogleTest application directly:
```sh
./build/bin/neko_test
```

## Shadertoy Converter

NekoDSL also includes a robust Python utility `to_shadertoy.py` to seamlessly convert your NekoDSL compiled fragment shaders into valid Shadertoy GLSL code natively. It invokes `SPIRV-Cross` to transpile and adapt uniform inputs.

**Usage:**
```sh
python to_shadertoy.py <your_shader.spv>
```

The tool will print directly to `stdout`. It adapts layout qualifiers, strips Vulkan semantics, and converts valid input bindings (like textures/samplers or UBOs) out into `iTime`, `iChannelX`, and injects `fragColor`/`fragCoord` into a standard `mainImage` function natively!
